# 数据过滤软件
一个用于处理txt文件中标签数据的GUI应用程序。
软件采用多页面设计，主页面提供功能入口，点击功能按钮可跳转到对应的功能页面。
## 功能特性
1. **数据标签过滤**
   - 选择包含txt文件的源文件夹
   - 自动扫描所有txt文件（不显示列表）
   - 支持保存文件夹位置，下次启动自动加载
   - **修改标签**：支持多种映射方式
     - 简单映射：`1->3,2->6` (多个映射用逗号分隔)
     - 范围映射：`1-5->10` (将1到5都映射为10)
     - 批量映射：`1,2,3->5` (将1,2,3都映射为5)
     - 混合使用：`1->3,2-4->6,5,6->10`
   - **删除标签**：支持删除指定标签
     - 格式：`1,2,3` (多个标签用逗号分隔)
     - 示例：删除标签1、2、3
2. **标签格式转换**
   - 支持TXT (YOLO) ↔ XML (Pascal VOC)
   - 支持COCO JSON ↔ XML (Pascal VOC)
   - 支持YOLO TXT ↔ COCO JSON
   - 支持类别映射：`0:cat,1:door`
   - 结果保存到指定输出文件夹，保护原始标注
3. **数据集分类**
   - 自动识别数据集类别和文件
   - 按比例划分训练集、验证集、测试集
   - 生成YOLO格式的YAML配置文件
   - 创建对应的txt文件用于模型训练
## 使用方法

### 安装

本软件主要使用Python标准库，但某些高级功能需要额外依赖：

```bash
# 安装所有依赖
pip install -r requirements.txt
```

### exe可执行文件打包

如果您想将程序打包为独立的exe文件，无需安装Python环境即可运行：

#### 方法1：一键打包（推荐）

```bash
# Windows用户双击运行批处理文件
build_exe.bat

# 或使用Python脚本
python simple_build.py
```

#### 方法2：自定义打包

```bash
# 安装PyInstaller
pip install pyinstaller

# 使用详细配置打包
python build_exe.py
```

打包完成后，在 `dist/` 目录下会生成 `数据过滤软件.exe` 文件。

**注意事项**：
- 打包过程需要一些时间，请耐心等待
- 生成的exe文件较大（约100-200MB），包含所有依赖
- exe文件可以直接在其他Windows电脑上运行，无需安装Python

### 运行

```bash
# Python源码运行
python data_filter.py

# 或直接运行exe文件
./dist/数据过滤软件.exe
```

### 操作步骤

#### 主页面操作
1. **启动软件**
   - 运行程序后，首先显示主页面
   - 主页面显示三个功能按钮：
     - 数据标签过滤
     - 标签格式转换
     - 数据集分类

#### 数据标签过滤
2. **进入数据标签过滤**
   - 点击主页面上的"数据标签过滤"按钮
   - 跳转到数据标签过滤页面

3. **选择源文件夹**
   - 点击"浏览"按钮选择包含txt文件的源文件夹
   - 或直接在输入框中输入文件夹路径
   - 点击"加载文件"按钮加载所有txt文件

4. **选择保存文件夹（可选）**
   - 点击保存文件夹区域的"浏览"按钮选择保存位置
   - 如果不设置，将直接修改源文件（会有确认提示）

5. **修改标签**
   - 在"标签映射"输入框中输入映射规则
   - 支持格式：
     - 简单映射：`1->3,2->6`
     - 范围映射：`1-5->10` (将1到5都改为10)
     - 批量映射：`1,2,3->5` (将1,2,3都改为5)
     - 混合使用：`1->3,2-4->6,5,6->10`
   - 点击"执行修改"按钮

6. **删除标签**
   - 在"要删除的标签"输入框中输入要删除的标签，格式：`标签1,标签2`
   - 多个标签用逗号分隔，例如：`1,2,3`
   - 点击"执行删除"按钮

#### 标签格式转换
7. **进入标签格式转换**
   - 点击主页面上的"标签格式转换"按钮
   - 跳转到标签格式转换页面

8. **选择转换方向**
   - TXT (YOLO) ↔ XML (Pascal VOC)
   - COCO JSON ↔ XML (Pascal VOC)
   - YOLO TXT ↔ COCO JSON

9. **设置路径和映射**
   - 标注文件夹：选择源标注文件所在目录
   - 图片文件夹：TXT→XML和YOLO→COCO转换时需要
   - 输出文件夹：必选，保存转换结果
   - 类别映射（可选）：`0:cat,1:door`
   - 点击"开始转换"

#### 数据集分类
10. **进入数据集分类**
    - 点击主页面上的"数据集分类"按钮
    - 跳转到数据集分类页面

11. **选择数据集路径**
    - 数据集根目录：包含images/和labels/文件夹
    - 输出目录：保存配置文件的位置

12. **设置划分比例**
    - 训练集比例：默认0.7
    - 验证集比例：默认0.2
    - 测试集比例：默认0.1
    - 随机种子：保证可重现划分

13. **生成配置**
    - 点击"扫描数据集"查看数据结构
    - 点击"生成配置文件"创建YAML和划分文件

#### 数据集展示
14. **进入数据集展示**
    - 点击主页面上的"数据集展示"按钮
    - 跳转到数据集展示页面

15. **选择数据集路径**
    - 数据集根目录：包含images/和labels/文件夹

16. **分析数据集**
    - 点击"分析数据集"扫描所有图片和标注文件
    - 提取类别、bbox尺寸、图片尺寸等统计信息

17. **选择图表类型**
    - 类别分布柱状图：显示各类别目标数量
    - 目标尺寸统计饼图：小中大目标占比
    - bbox尺寸散点图：宽度-高度分布
    - bbox面积分布：面积直方图
    - bbox中心点热力图：目标位置分布
    - 每图目标数统计：每张图片目标数量分布
    - 图片尺寸分布：图片尺寸散点图

18. **查看图表**
    - 选择图表类型后自动生成并显示
    - 支持交互式查看和分析

19. **返回主页**
    - 在任意功能页面点击"🏠 返回主页"按钮可返回主页面

## 注意事项

- 如果设置了保存文件夹，处理后的文件将保存到指定文件夹，源文件不会被修改
- 如果未设置保存文件夹，将直接修改源文件（会有确认提示）
- 标签匹配使用正则表达式，会匹配文件中的所有数字
- 建议在处理重要文件前先备份
- 配置文件保存在程序目录下的`config.json`，包含源文件夹和保存文件夹路径
